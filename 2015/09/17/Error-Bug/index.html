开发中常见的Error、Bug

  • 开发中常见的Error、Bug

常见的错误

1.复制布局文件时，有自定义view，要注意改其包名。

2.String类型方法传非String类型的值

错误原因： 不能把int的值传给String类型的方法.
解决办法：加一个空字符””

3.项目与Lib中的定义的属性相冲突

解决办法：
在Manifest.xml的application标签下添加tools:replace=”android:icon, android:theme”（多个属性用,隔开，并且记住在manifest根标签上加入xmlns:tools=”http://schemas.android.com/tools”，否则会找不到namespace）

4.自定义View没有重写构造函数

解决办法：重写三个构造函数

5.使用自带的控件报错

错误原因：没有使用V4包里的viewpager 而是使用了自带的viewpager
即XML文件中定义了ViewPager而不是android.support.v4.view.ViewPager

解决方法：将ViewPager改成android.support.v4.view.ViewPager（可预览）

6.关于getItemViewType报数组越界的问题

错误原因

getItemViewType的值必须小于getItemViewTypeCount

  1. JAVA_HOME错误

原因
AS使用了自带的JDK
解决方法
Go to File -> Other Settings -> Default Project Structure

JAVA_HOME错误

  1. 在非UI线程里操作UI

原因
在非UI线程里操作UI
解决方法
创建Handler，在非UI线程中利用handler对象发送消息。从而更新UI。

BUG

1.ActivityforResult没效果

错误原因
1.请求码不一样
2.没有调用startActivityForResult方法
3.请求码不大于0

8.自定义View的原控件ID与使用时ID相同，报转型错误

示例
CustomActionBar里面有TextView id为tvname;使用CustomActionBar时，命名为tvName会报错。

ScrollView嵌套ListView导致的BUG

ListView顶走上方的内容

解决方法
ScrollView外层添加android:descendantFocusability=”blocksDescendants”属性
让ListView失去焦点

问题解决
焦点参考

ListView只显示一行数据

解决方法
在获取数据后,先计算ListView高度,然后再更新adapter。

NotSerializableException

原因
1.实体类没有implement Serializable接口。
2.实现了Serializable接口,但更新代码后,没有清理之前的数据。
解决方法
实现Serializable接口、清除app数据。

Cannot find System Java Compiler. Ensure that you have installed a JDK (not just a JRE) and configured your JAVA_HOME system variable to point to the according directory.

安装应用失败报错

install_canceled_by_user
可能原因
1.手机并未允许安装未知来源应用
2.手机处于锁屏状态
3.某个Activity设置了特定的taskAffinity属性。
