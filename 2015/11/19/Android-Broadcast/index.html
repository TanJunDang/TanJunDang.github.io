四大组件之Broadcast

  • Broadcast

Broadcast的理解

  • 当收到某系统消息时，可以做相应的处理。
  • 动态注册：程序启动后才有效。
  • 静态注册：不需要启动程序就有效。

动态广播实际开发

  • 服务器下发新消息时，通知MainActivity更改UI
  1. 继承BroadcastReceiver
  2. 定义过滤的Action
  3. 添加回调接口
  4. 构造函数初始化接口
  5. 重写onReceive方法(调用接口的方法)
  6. 实现接口、动态注册接口(LocalBroadcastManager类)
  7. 在获取到服务器数据的地方发送广播
  8. 在ondestory里取消广播

静态广播实际开发

  • 监听系统网络状态改变,从而做相应的处理。

同上,注册的时候用registerReceiver、unregisterReceiver,用LocalBroadcastManager无效。

Fragment里注册广播注意事项

  • 不能在OnCreateView里面注册,要在onCreate方法里面注册
  • 记得在ondestory方法里注销广播
