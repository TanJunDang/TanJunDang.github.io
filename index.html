Socket学习笔记

Socket基础知识

Socket用于客户端与服务端间进行网络通信。包括连接服务器、发送接收数据、关闭网络连接的功能。

Socket通信步骤

1.连接服务器 - 连接方式分构造函数、connect方法两种。
2.发送和接收数据 - 涉及Out(In)putStream、BufferWriter(Reader)读取流的方式。
3.关闭连接 - 调用各种流的close方法。

Socket的连接方式

Socket连接服务端的方式分为构造函数、connect方法两种。

示例 - 构造函数方式

Socket(String host , int port)//参数分别服务端主机地址、端口

示例 - connect方法

使用connect方法必须通过Socket的无参构造函数来创建对象。
connect(SocketAddress add)//参数为SocketAddress,该类封装了主机地址以及端口
connect(SocketAddress add ,int timeout)//第二个参数是超时参数、默认为0

Socket常用的方法

Socket常用方法 说明
isClosed() 判断Socket对象是否处于关闭状态
true - 关闭 false - 连接
isConnected() 判断Socket对象是否成功连接过
(!isClosed() && isConnected()) 判断socket对象是否处于连接状态
getInetAddress() 返回一个InetAddress对象，通过该对象可获取服务器IP，Port等信息
getLocalAddress() 返回一个InetAddress对象，通过该对象可获取本机的IP，port等信息
getInputStream()、getOutputStream() 输入输出流，用于接收和发送数据
setReuseAddress(boolean on) 当多个socket绑定一个端口时，需要调用setReuseAddress(true),否则会出Bind异常
setSoLinger(boolean on,int linger) 当socket关闭时，若还没有发送完数据，则阻塞linger秒，从而尽量把数据发完
setTimeOut(int timeout) 设置读取数据超时，单位毫秒
setKeepAlive(boolean on) 该方法用于检测Socket是否处于活动状态，客户端Socket每隔一段时间利用空闲的时间向服务器发送一个检测数据包，如果服务器未响应该数据包，在几分钟后，客户端socket再发一个数据包，若服务器还没响应，则客户端Socket关闭。

ScrollView学习笔记

ScrollView基础知识

理解
该View是一个ViewGroup，其子类只能有一个。通常用RelativeLayout或者LinearLayout做其子类。
作用
使子类可以滚动。当一个页面内容过多的时候，手机屏幕不能显示完全，就需要通过该View来实现滚动。

ScrollView常用属性 作用 值 示例
android:overScrollMode=”never” 设置滑动到边缘时，是否显示阴影 always - 显示阴影(默认)
never - 不显示阴影
ifContentScrolls - 仅当视图内容大于容器时，允许滚动
android:fillViewport=”true” 设置子内容是否可填满ScrollView true - 可填满ScrollView
false - 不可填满		 不添加该属性时，子ViewLinearLayout设置width、height为match_patent时，是无效的。
android:scrollbars=”none” 设置滚动条方向 none - 不显示
vertical - 垂直滚动条
horizontal - 水平滚动条
android:nestedScrollingEnabled=”false” 设置是否启用嵌套滑动 false - 不启动嵌套滑动(无法滑动，交给父View滑动)
true - 启动嵌套滑动

TabLayout笔记

TabLayout基础知识

理解
TabLayout通常与ViewPager联合使用。
TabLayout主要用于设置tab标题、关联ViewPager、设置自定义Tab、设置tab选中监听器。
ViewPager主要用于创建FragmentPagerAdapter、设置Adapter。（跟ListView用法类似）
使用
1.在XML里设置TabLayout跟ViewPager。通常纯文本的Tab在顶部，包含Icon的Tab在底部。
2.创建Tab标题数组以及Fragment的List集合。
3.创建FragmentPagerAdapter，重写getCount、getItem、getPageTitle等方法。
4.ViewPager创建Adapter并调用setAdapter方法。
5.TabLayout使用setupWithViewPager进行关联。(如果是自定义Tab，还需要调用setTabsFromPagerAdapter)

ActionBar笔记

ActionBar基本知识

1.Android 3.0(API level 11)版本之后，所有使用默认主题的Activity都包含有Actionbar。
2.如果需要支持Android 2.0(API level 7)版本，需要修改mainfest中的minSdkVersion版本。
3.如果需要自定义ActionBar，则先设置Activity的Theme为NoActionBar(需要设置name/parent)，然后通过添加ToolBar来实现。
4.values、values-xx的理解。

  • values表示所有API使用该文件。
  • values-xx表示XXAPI起，使用该文件。
