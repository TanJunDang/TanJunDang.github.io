WebView学习笔记

WebView基础知识

理解
webView可以理解为一个加载url页面的控件。可与js交互。
Android与JS的交互需要通过接口实例化的对象进行。
webView调用loadUrl的方式
1.mWebView.loadUrl(“javascript：test()”);// 调用js中的test函数
2.mWebView.loadUrl(“file:///android_asset/index.html”);// 调用本地assets目录下的html文件，也可以直接调用网上的html文件。
重定向是什么？加载网页的时候，网页设置了重定向，即你要跳转的url被重新修改，使你跳转的新的URL。例如你要跳转www.baidu.com，百度会重定向使你跳转www.m.baidu.com
webView相关的API有WebChromeClient、WebViewClient、WebSetting三个。详细见下表

API 作用 示例
WebChromeClient 获取网页中的信息(标题、图标)、加载进度、处理js对话框等 重写onReceivedTitle获取网页标题
WebViewClient 处理WebView的通知、请求事件
如果不创建该对象，则会调用系统自带的浏览器启动新的页面 		重写shouldOverrideUrlLoading由WebView处理新的url
WebSettings 配置webView、设置自适应屏幕、支持缩放、支持js交互等 setBuiltInZoomControls设置支持缩放
WebView 加载url、跳转页面顶部、设置是否显示滚动条 setScrollY(0)回到页面顶部
WebChromeClient常用方法 作用
onReceivedTitle(WebView view, String title) 获取网页中的标题
onReceivedIcon(WebView view, Bitmap icon) 获取网页中的图标
onProgressChanged(WebView view,int newProgress) 获取网页加载的进度
onJsAlert(WebView view,String url,String message,JsResult result) 处理js中的Alert对话框
WebViewClient常用方法 作用
shouldOverrideUrlLoading(WebView view,String url) 加载新页面时的回调
返回true 页面任何点击无效，需要对WebView、url进行处理，通常是调用loadUrl加载页面，与返回false没什么不同，所以一般不重写 。或者需要通过浏览器来启动时，则需要返回true
返回false(默认) 则调用webView处理新的页面
onPageStared(WebView view,String url) 页面开始加载时的回调
onPageFinished(WebView view,String url,Bitmap favicon) 页面加载结束的回调
onScaleChanged(WebView view,float oldScale,float newScale) 缩放页面时的回调
onReceivedError(WebView view,int errorCode,String description,String failingUrl) 网页错误的时候回调，可获取错误信息
WebSettings常用方法 作用
setUseWideViewPort(true)
setLoadWithOverviewMode(true)		 将图片调整到适合webview的大小
缩放至屏幕的大小
两句结合使用使页面自适应屏幕
setBuiltInZoomControls(boolean) 设置是否支持缩放
setSupportZoom(boolean) 这方法好像没什么用，但是如果设置了false，setBuiltInZoomControls变无效
webSettings.setJavaScriptEnabled(true)
webSettings.setDefaultTextEncodingName(“UTF-8”)		 设置webview允许调用js

TabLayout笔记

TabLayout基础知识

理解
TabLayout通常与ViewPager联合使用。
TabLayout主要用于设置tab标题、关联ViewPager、设置自定义Tab、设置tab选中监听器。
ViewPager主要用于创建FragmentPagerAdapter、设置Adapter。（跟ListView用法类似）
使用
1.在XML里设置TabLayout跟ViewPager。通常纯文本的Tab在顶部，包含Icon的Tab在底部。
2.创建Tab标题数组以及Fragment的List集合。
3.创建FragmentPagerAdapter，重写getCount、getItem、getPageTitle等方法。
4.ViewPager创建Adapter并调用setAdapter方法。
5.TabLayout使用setupWithViewPager进行关联。(如果是自定义Tab，还需要调用setTabsFromPagerAdapter)

自定义View笔记

自定义View的知识(基础)

理解
~自定义View是在画布上面画的，所以要区分清楚画布的大小跟View的大小。

常见问题 理解
自定义View的方式有哪些？ 继承View、继承ViewGroup、继承ViewGroup的组合View
自定义View的方法的执行顺序是什么？ onMeasure->onSizeChange->onLayout->onDraw
什么是测量值？ 即在xml文件设定的宽高值。当宽高为wrap时，默认大小为屏幕大小。
如何理解onMeasure方法？ 决定画布的大小
自定义View，默认宽高为warp_content跟match_parent效果一致，如何解决？ 重写onMeasure、处理Mode为AT_MOST的情况
如何获取滑动最小距离？ ViewConfiguration.get(this).getScaledDoubleTapSlop()
