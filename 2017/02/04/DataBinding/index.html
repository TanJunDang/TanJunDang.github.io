DataBinding的理解

DataBinding是什么，有什么用处？
DataBinding是15年I/O大会推出的新技术，主要作用是减少FindId，setListener代码的出现、数据双向绑定(数据变化自动更新UI)、提高性能等。

DataBinding无法生成的原因

布局没有layout设置根标签

找不到BR类

项目中使用过butterknife，两者有冲突。导致无法生成BR类
解决方法

1.移除gradle中的apply plugin: ‘com.neenbedankt.android-apt’
2.将apt改成annotationProcessor

数据绑定实现自动更新UI

理解
就是将Data跟View进行绑定，当Data变化时自动更新UI，View不需要像以前那样重新setData。

Observable绑定方式

步骤
1.实体继承BaseObservable。
2.set方法添加notifyPropertyChanged通知方法。
3.get方法添加Bindable注解。

DataBinding遇到的坑

使用ObservableField出错

data binding error **msg:Could not find accessor
原因：ObservableField声明的变量、方法必须为public final

内存超标

Error:Execution failed for task ‘:app:mergeDebugResources’.
GC overhead limit exceeded
在local.properties、gradle.properties文件添加下面代码
org.gradle.jvmargs=-Xmx4096m
