WebView基础知识

理解
webView可以理解为一个加载url页面的控件。可与js交互。
Android与JS的交互需要通过接口实例化的对象进行。
webView调用loadUrl的方式
1.mWebView.loadUrl(“javascript：test()”);// 调用js中的test函数
2.mWebView.loadUrl(“file:///android_asset/index.html”);// 加载本地html，在app/src/main下新建一个文件夹，取名assets（不能是res目录下新建，否则无法加载）
重定向是什么？加载网页的时候，网页设置了重定向，即你要跳转的url被重新修改，使你跳转的新的URL。例如你要跳转www.baidu.com，百度会重定向使你跳转www.m.baidu.com
webView相关的API有WebChromeClient、WebViewClient、WebSetting三个。详细见下表

API 作用 示例
WebChromeClient 获取网页中的信息(标题、图标)、加载进度、处理js对话框等 重写onReceivedTitle获取网页标题
WebViewClient 处理WebView的通知、请求事件
如果不创建该对象，则会调用系统自带的浏览器启动新的页面 		重写shouldOverrideUrlLoading由WebView处理新的url
WebSettings 配置webView、设置自适应屏幕、支持缩放、支持js交互等 setBuiltInZoomControls设置支持缩放
WebView 加载url、跳转页面顶部、设置是否显示滚动条 setScrollY(0)回到页面顶部
WebChromeClient常用方法 作用
onReceivedTitle(WebView view, String title) 获取网页中的标题
onReceivedIcon(WebView view, Bitmap icon) 获取网页中的图标
onProgressChanged(WebView view,int newProgress) 获取网页加载的进度
onJsAlert(WebView view,String url,String message,JsResult result) 处理js中的Alert对话框
WebViewClient常用方法 作用
shouldOverrideUrlLoading(WebView view,String url) 加载新页面时的回调
返回true 页面任何点击无效，需要对WebView、url进行处理，通常是调用loadUrl加载页面，与返回false没什么不同，所以一般不重写 。或者需要通过浏览器来启动时，则需要返回true
返回false(默认) 则调用webView处理新的页面
onPageStared(WebView view,String url) 页面开始加载时的回调
onPageFinished(WebView view,String url,Bitmap favicon) 页面加载结束的回调
onScaleChanged(WebView view,float oldScale,float newScale) 缩放页面时的回调
onReceivedError(WebView view,int errorCode,String description,String failingUrl) 网页错误的时候回调，可获取错误信息
WebSettings常用方法 作用
webSettings.setUseWideViewPort(true);
webSettings.setLoadWithOverviewMode(true);		 将图片调整到适合webview的大小
缩放至屏幕的大小
两句结合使用使页面自适应屏幕
webSettings.setSupportZoom(boolean) 一般不用设置，默认为true，设置了false，setBuiltInZoomControls变无效
webSettings.setBuiltInZoomControls(boolean) 设置是否支持缩放
webSettings.setDisplayZoomControls(boolean) 设置是否显示缩放按钮（加减号）
webSettings.setJavaScriptEnabled(true)
webSettings.setDefaultTextEncodingName(“UTF-8”)		 设置webview允许调用js(默认是false)
webSettings.setDomStorageEnabled(true); 前端发ajax请求,没有数据返回，设置该句代码
webSettings.setBlockNetworkImage(false); false为允许网络图片数据的显示，true不允许显示网络图片（默认false）

WebView缓存

WebView缓存分成两种类型页面缓存、数据缓存。
页面缓存通常是由浏览器产生的。一般不处理。数据缓存分为AppCache、DOM Storage：两种。
AppCache：我们能够有选择的缓冲web浏览器中所有的东西，从页面、图片到脚本、css等等。 尤其在涉及到应用于网站的多个页面上的CSS和JavaScript文件的时候非常有用。其大小目前通常是5M。 在Android上需要手动开启（setAppCacheEnabled），并设置路径（setAppCachePath）和容量 （setAppCacheMaxSize），而Android中使用ApplicationCache.db来保存AppCache数据！
DOM Storage：存储一些简单的用key/value对即可解决的数据，根据作用范围的不同，有Session Storage和Local Storage两种，分别用于会话级别的存储（页面关闭即消失）和本地化存储（除非主动 删除，否则数据永远不会过期）。

CacheMode

LOAD_DEFAULT: 根据cache-control决定是否从网络上取数据。cache-control是什么
LOAD_CACHE_ONLY: 不使用网络，只读取本地缓存数据
LOAD_NO_CACHE: 不使用缓存，只从网络获取数据.
LOAD_CACHE_ELSE_NETWORK: 只要本地有缓存，都使用缓存中的数据。本地没有则从网上获取。

清除缓存的方法

webView.clearCache(true);
setting.setCacheMode(WebSettings.LOAD_NO_CACHE);
deleteDatabase(“WebView.db”);和deleteDatabase(“WebViewCache.db”);
webView.clearHistory();
webView.clearFormData();
getCacheDir().delete();
手动写delete方法，循环迭代删除缓存文件夹！
