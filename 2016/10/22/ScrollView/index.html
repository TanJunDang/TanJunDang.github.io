ScrollView基础知识

理解
该View是一个ViewGroup，其子类只能有一个。通常用RelativeLayout或者LinearLayout做其子类。
作用
使子类可以滚动。当一个页面内容过多的时候，手机屏幕不能显示完全，就需要通过该View来实现滚动。

ScrollView常用属性 作用 值 示例
android:overScrollMode=”never” 设置滑动到边缘时，是否显示阴影 always - 显示阴影(默认)
never - 不显示阴影
ifContentScrolls - 仅当视图内容大于容器时，允许滚动
android:fillViewport=”true” 设置子内容是否可填满ScrollView true - 可填满ScrollView
false - 不可填满		 不添加该属性时，子ViewLinearLayout设置width、height为match_patent时，是无效的。
android:scrollbars=”none” 设置滚动条方向 none - 不显示
vertical - 垂直滚动条
horizontal - 水平滚动条
android:nestedScrollingEnabled=”false” 设置是否启用嵌套滑动 false - 不启动嵌套滑动(无法滑动，交给父View滑动)
true - 启动嵌套滑动

滑动冲突

产生原因
当可滑动ViewGroup嵌套可滑动View(ListView、RecyclerView等)时，由于两者都有滑动属性，因此会导致滑动冲突，从而导致滑动卡顿、显示不全等现象。
