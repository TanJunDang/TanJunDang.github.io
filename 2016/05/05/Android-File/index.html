  • Android - File文件操作

Java文件的基础知识

理解 —— File是一个工具类,通过其对象来操作文件的增删改查

文件流的理解

理解 —— 在程序中要操作文件(读写)，必须先将文件转换成流，然后从流中读取(写入)数据。

1.字符流(Reader、Writer) — 读取(写入)相关的字符(String)，不需要缓冲区。

2.字节流(InputStream、OutputStream) — 读取(写入)相关的字节,需要先将数据读入(写入)缓冲区，然后再将缓冲区读入(写入)流中。

总结

1.字符流比字节流简便。
2.不推荐使用FileInputStream、FileInputStream，要用就结合FileChannel来用。

1.不要 ——> 直接创建文件
2.需要 ——> 判断文件的目录是否存在 && 判断是否文件夹 ——> 如果不存在，则在该路径下创建文件

Android中常用的一些目录

API - Context

getCacheDir() - 用于获取内部存储空间路径/data/data/包名/cache/
getFilesDir() - 用于获取内部存储空间路径/data/data/包名/files/

openFileInput(String name) - 打开应用程序私有目录下的的指定私有文件以读入数据，返回一个FileInputStream 对象。

openFileOutput(String name , int mode) - 打开应用程序私有目录下的的指定私有文件以写入数据，返回一个FileOutputStream 对象。如果文件不存在就创建这个文件。

fileList() - 列出程序所有私有文件夹下的所有私有文件。

deleteFile(String fileName) - 删除指定文件名的文件，成功返回true,失败返回false。

getDir(String name , int mode);//创建一个目录，返回一个File对象。

getExternalFilesDir(String name) - 用于获取外部存储空间名称为name的目录的路径(/storage/emulated/0/Android/data/package_name/Sam/) 有些设备的部分路径不是/storage/emulated/0而是/mnt/sdcard

MODE_PRIVATE 默认模式，文件只可以被调用该方法的应用程序访问
1.MODE_APPEND 如果文件已存在就向该文件的末尾继续写入数据，而不是覆盖原来的数据。
2.MODE_WORLD_READABLE 赋予所有的应用程序对该文件读的权限。
3.MODE_WORLD_WRITEABLE 赋予所有的应用程序对该文件写的权限。

API - Environment

getExternalStorageDirectory() - 获取Android根目录 storage/emulated/0

getExternalStoragePublicDirectory(String type) - 获取Android系统中的某目录 /storage/emulated/0/xxxx 参数为Environment的常量

getExternalStorageState() - 获取外部存储设备的当前状态

getDataDirectory() - 获取Android的数据源目录 /data

getRootDirectory() - 获取 Android 的根目录 /android

Environment常量
1.DIRECTORY_DCIM - 系统的DCIM目录
2.DIRECTORY_DOWNLOADS - 系统的Downloads目录
3.DIRECTORY_ALARMS - 系统的目录Alarms目录
4.MEDIA_MOUNTED - SD卡插入且可读写
….

文件内外部存储

Android内部存储（用户不可操作）

文件只能被自己的应用访问,并且当应用被卸载时,数据也会被销毁。Shared Preferences和SQLite数据库都是存储在内部存储空间上的。所有应用程序的内部存储的私有文件都放在根目录的data/data/package_name/files/下.

Android外部存储（用户可操作）

文件可以被其他应用访问,当应用被卸载时,数据不会被销毁。所有应用程序的外部存储的私有文件都放在根目录的Android/data/package_name/下，目录形式为/Android/data/package_name/

总结

数据写入文件的基本流程：(writeObject、ObjectOutputStream)
1.用File类确定文件夹的路径（进行必要的判断和创建）
2.用File类确定文件是否存在，存在则删除。
3.用ObjectOutputStream向文件写入数据

读取数据的基本流程：(readObject、ObjectInputStream)
1.用File确定文件夹路径
2.用File确定文件
3.判断文件是否存在,用ObjectInputStream读取数据
