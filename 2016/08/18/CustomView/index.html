自定义View的知识(基础)

理解
~自定义View是在画布上面画的，所以要区分清楚画布的大小跟View的大小。

常见问题 理解
自定义View的方式有哪些？ 继承View、继承ViewGroup、继承ViewGroup的组合View
自定义View的方法的执行顺序是什么？ onMeasure->onSizeChange->onLayout->onDraw
什么是测量值？ 即在xml文件设定的宽高值。当宽高为wrap时，默认大小为屏幕大小。
如何理解onMeasure方法？ 决定画布的大小
自定义View，默认宽高为warp_content跟match_parent效果一致，如何解决？ 重写onMeasure、处理Mode为AT_MOST的情况
如何获取滑动最小距离？ ViewConfiguration.get(this).getScaledDoubleTapSlop()

组合View

思路

  1. xml布局文件添加已有的控件，例如(ImageView跟Button)。
  2. 创建java文件继承包含这些控件的布局，例如(RelativeLayout)。
  3. 重写其构造方法(加载xml布局文件，利用通过LayoutInflate)。
  4. 在其构造方法里获取自定义属性。(可选,在values文件夹创建一个attrs的xml文件)
  5. 对属性值进行处理。

总结

为自定义View添加自定义属性

1.在values目录下的attrs文件中，创建declare-styleable标签，其name属性是控件名字。
2.创建attr子标签，name属性是属性名、format指明该属性的值的类型。

如果定义属性时，属性添加了前缀，则使用时需要设置命名空间
xmlns:tjd=”http://schemas.android.com/apk/res-auto“ //包名指整个项目的包名

通过该命名空间调用属性 如 tjd:属性名
tjd:actionbarTitle=”TANJUNDANG”

属性里的值可以通过enum标签来实现

类型 说明
reference 表示引用
string 表示字符串
color 表示颜色值
dimension 表示尺寸值
boolean 表示布尔值
integer 表示整型值
float 表示浮点值
fraction 表示百分数
enum 表示枚举值
flag 表示位运算

完全自定义View(进阶)

步骤 作用 实现
1 属性 可以通过xml自定义控件的属性，然后通过TypedArray来进行使用。
2 绘图 通过重写onDraw方法控制View在屏幕上的渲染效果。
3 测量 通过重写onMeasure方法来对控件进行测量。
4 布局 通过重写onLayout方法来对View进行布局，继承ViewGroup时必须用到。
5 交互 通过重写onTouchEvent方法或者使用手势来控制用户的交互。
6 状态的保存 为了避免配置改变时丢失View状态，通过重写onSaveInstanceState，onRestoreInstanceState方法来保存和恢复状态。

为什么要重写onMeasure

1.默认情况下Customview设置宽高为warp、match时，都会填满父容器
2.重写onMeasure可以为CustomView设置默认值。

处理 结果
不重写onMeasure Customview的宽高为warp_content时，view填满父容器
重写onMeasure 可实现Customview为宽高warp时，显示默认值，而不是填满父容器

注意

自定义View的layout文件根标签为match_parent导致CustomView设置。warp_content时，会填满整个布局文件。

根据需求重写OnMeasure

需求 是否需要重写
继承View时 需要
继承ViewGroup时 非必需

怎么重写onMeasure

重写onMeasure方法的步骤
1.利用MeasureSpec类计算size、mode。
2.通过setMeasuredDimension方法设置计算结果。

MeasureSpec参数 说明
EXACTLY 当父控件宽高设置为match_parent或者固定值时，CustomView的宽高应该尽量大
AT_MOST 当父控件宽高设置为warp_content时,CustomView的宽高应该是默认值
UNSPECIFIED 不对子View做限制

实例 - 自定义请求View

作用：

1.请求的时候显示进度条加载。
2.请求失败时显示失败页面，点击页面可以重新请求。(显示失败的时候才可以点击)

实现：

1.继承RelativeLayout。

2.布局中使用谷歌的ProgressBar、ImageView、TextView等控件。

3.自定义pgbType属性,用于决定ProgressBar的显示类型，并在构造函数中获取。

4.由于请求时，不能点击下层的按钮。所以需要设置布局文件clickable为false。
