RecyclerView的个人理解

RecycleView的用法

基本理解

RecyclerView.ViewHolder：承载Item视图的子视图；
RecyclerView.LayoutManager：负责Item视图的布局；
RecyclerView.ItemDecoration：为每个Item视图添加子视图，在Demo中被用来绘制Divider；
RecyclerView.ItemAnimator：负责添加、删除数据时的动画效果；
RecyclerView.Adapter：数据集合，为每个Item创建视图；

1.onCreateViewHolder
设置监听事件、加载布局、获取缓存数据

2.onBindViewHolder
设置显示数据、设置缓存TAG

3.RecyclerView.ViewHolder
Find相关的ID

RecyclerView.ItemDecoration - 分割线

理解
ItemDecoration用于绘制RecyclerView的Divider
ItemDecoration涉及三个API。分别是onDraw、onDrawOver、getItemOffsets。
待重写方法
onDraw(Canvas c, RecyclerView parent, State state)//一般不重写onDraw而是onDrawOver。
onDrawOver(Canvas c, RecyclerView parent, State state)//计算分割线的宽高
getItemOffsets(Rect outRect, View view, RecyclerView parent, State state)//设置Item的外边距，但不影响Divider的长度。
重写注意
重写onDrawOver的时候，涉及RecyclerView、Item、Divider三个View的宽高处理。

RecyclerView添加Header跟Footer

理解
由于RecyclerView没有类似ListView中的addFooter、addHeader等方法。所以要添加头部、底部需要结合getViewType来添加。
使用
1.Holder的构造函数新增viewtype参数，根据viewtype来FindID
2.然后结合getItemViewType()方法来返回相应的Type
3.onCreateViewHolder根据ViewType来加载相应的布局

方法一

compile ‘com.android.support:recyclerview-v7:23.2.1’

方法二

由于RecyclerView Item的高度由LinearLayoutManager决定，所以添加一个自适应的LinearLayoutOutManager。

RecyclerView滑动卡顿

滑动卡顿主要是因为与ScrollView的滑动有冲突。所以禁止RecyclerView滑动或者ScrollView既可。

方法一

recyclerView调用setNestedScrollingEnabled方法禁止滑动。

PS
由于RecyclerView的XML属性里不包括android:nestedScrollingEnabled，所以在RecyclerView里使用android:nestedScrollingEnabled=”false”代码会崩溃。

方法二

重写LinearLayoutManager的canScrollVertical方法，使其返回false。
